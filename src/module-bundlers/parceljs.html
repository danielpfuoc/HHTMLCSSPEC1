ParcelJS es un paquete de aplicaciones web, diferenciado por su experiencia de desarrollador. Ofrece un rendimiento increíblemente rápido utilizando el procesamiento multinúcleo y no requiere configuración. Con parcel, no tiene que configurar ninguna configuración ni ningún cargador para CSS, JS, activos e.t.c.

Se trata de un compilador para todo su código, independientemente del idioma o cadena de herramientas.

Lo que es toomar todos sus archivos y dependencias, los transforma y los combina en un conjunto más pequeño de archivos de salida que pueden usarse para ejecutar su código.

Admite muchos idiomas y tipos de archivos diferentes, desde tecnologías web como HTML, CSS y JavaScript, hasta idiomas de nivel inferior como Rust y cualquier cosa que se compile en WebAssembly (WASM), hasta activos como imágenes, fuentes, videos, y más.

Hace que su código sea portátil. Puede crear su código para diferentes entornos, para la web para su servidor o para una aplicación. Incluso puede crear múltiples objetivos a la vez y hacer que se actualicen en vivo a medida que realiza cambios.

El paquete es rápido y predecible. Compila todos sus archivos de forma aislada en paralelo dentro de los trabajadores, almacenándolos en caché a medida que avanza. Los cachés son estables en todas las máquinas y solo se ven afectados por los archivos y las configuraciones dentro de su proyecto (a menos que desee pasar variables de entorno específicas).

¿Por qué deberías usar Parcel?

  • Configuración cero
  • Rendimiento
  • Reemplazo de módulos en caliente
  • División de código
  • Admite muchos tipos de activos
